# AIツール利用と判断軸の検証

## 目的

このファイルは、ITパスポートの問題演習で ChatGPT / Codex などの回答を使うときに、回答をそのまま採用してよいかを検証するためのメモです。

AIツールの機能、料金、使い方そのものを検証する場所ではありません。対象は、テストの問題を解くときの正答根拠、選択肢の切り方、試験側へ戻してよい判断軸です。

## 検証する問い

- GPT回答が合っていそうに見えるとき、どこを確認すれば採用してよいか
- 四択が全部それっぽく見えるとき、どの差分を見れば切れるか
- 長い説明のうち、試験側へ戻すべき部分はどこか
- 一般論ではなく、テストで再利用できる形になっているか

## 採用してよい条件

GPT回答は、次の条件を満たすときに採用しやすいです。

- 問題文の手がかり語と正解が対応している
- 正解だけでなく、他の選択肢を切る理由がある
- 用語の意味が、試験で問われる定義へ戻っている
- 最後に短い判別フレーズへ圧縮できる

逆に、説明がもっともらしくても、問題文のどの語からその答えになるかが説明できない場合は、まだ検証中として扱います。

## 今回の例

DX の問題では、見るべき核心は次の語でした。

- `ITやAIを活用`
- `ビジネスモデルの刷新`
- `新たな付加価値`

特に `ビジネスモデルの刷新` が強い手がかりです。

この場合、試験で使う判別フレーズは次の形にできます。

- 看板 = デジタルサイネージ
- 格差 = デジタルディバイド
- 変革 = デジタルトランスフォーメーション
- 生まれつき世代 = デジタルネイティブ

この形まで落ちれば、AI回答は「読んで終わり」ではなく、次回の四択で選択肢を切る材料として使えます。

## 残す判断

- `02_検証` に残すのは、AIツールの使い方そのものではなく、問題演習で回答を検証する手順
- GPT回答は正答そのものではなく、問題文の手がかり語、選択肢の差分、短い判別フレーズへ加工してから採用する
- 試験側へ戻すのは、カテゴリ別知識、用語の住み分け、選択肢を切る判断軸だけにする
