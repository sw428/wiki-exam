# 一瞬フレーズ学習法の検証

## 目的

このファイルは、ITパスポートの用語暗記について、GPTとの対話から見えた学習方法を検証するためのメモです。

元ログは全文保存せず、資格勉強に関係する判断だけを残します。

## 検証した問い

- 用語をそのまま丸暗記するより、問題の箱を作る方が覚えやすいのか
- 1日に覚えられる量は、単語数だけで決まるのか
- 長時間勉強しても意味があるのか
- ノートにまとめる前に、情報の粒度を加工する作業自体が記憶に効くのか

## 採用した見方

ITパスポートでは、用語を長い説明文として覚えるより、問題文や選択肢を見た瞬間に使える短い変換ルールとして残す方が実用的です。

例:

- `VoIP` = ネット電話
- `NTP` = 時刻合わせ
- `NFC` = かざして通信
- `MVNO` = 格安SIM
- デジタル署名 = 改ざん検知
- 暗号化 = 中身を隠す
- 商標 = ロゴ
- 特許 = 発明

これは単語を保存しているというより、選択肢を切るための判定軸を作っている状態です。

## 分かったこと

### 丸暗記の1個と加工済みの1個は重さが違う

同じ1用語でも、文字列のまま覚える場合と、意味・イメージ・判定軸に加工して覚える場合では負荷が違います。

例:

- `MTBSI = Mean Time Between Service Incidents`
- `MTBSI = サービスが止まる間隔`
- `MTBSI = 次に店が閉まるまでの平均時間。長いほど止まりにくい`

下に行くほど、試験で使える形に近づきます。

### 覚える量ではなく、作れる判定軸の数で見る

「1日何単語覚えられるか」より、「1日にいくつ使える判定軸を作れたか」で見る方が実感に近いです。

ITパスポートでは、次のような判定装置を増やす方が点につながります。

- 停電直後 = UPS
- 見たら検索 = ドライブバイダウンロード
- ネット電話 = VoIP
- 時刻合わせ = NTP
- 発明 = 特許
- ロゴ = 商標
- 売り方 = 特定商取引法
- 改ざん検知 = デジタル署名
- 中身を隠す = 暗号化

### 長時間学習は意味があるが、新規暗記だけだと効率が落ちる

長時間勉強する場合、新しい用語を詰め続けるより、次の処理に分ける方がよさそうです。

1. 問題を解く
2. 間違えた問題を確認する
3. 一瞬フレーズに変える
4. 似た用語と比べる
5. もう一回思い出す
6. 寝る前に短いフレーズだけ見る

新規を増やすだけではなく、思い出す時間と加工する時間を確保するのが重要です。

## 残す判断

- ITパスポートは、用語を百科事典のように暗記するより、用語の箱分けゲームとして扱う
- 1問につき、長い説明を残す前に「一瞬フレーズ」を作る
- 迷った問題、間違えた問題、似ていて混ざる問題だけを深く処理する
- 1日に深く作る判定軸は、10から20個程度でも十分多い
- 問題数はもっと回してよいが、全部を同じ濃度で覚えようとしない
