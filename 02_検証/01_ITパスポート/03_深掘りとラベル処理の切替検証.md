# 深掘りとラベル処理の切替検証

## 目的

このファイルは、ITパスポートの問題を解くときに、社会構造や言葉の背景まで深く考えるモードと、試験上のラベルを素早く選ぶモードを切り替えるための検証メモです。

ITパスポートは IT だけでなく、会社、契約、お金、管理、組織、社会で共有される抽象語彙を広く扱います。そのため、用語の背景を考えること自体は理解の助けになります。

ただし、試験中は深い社会考察ではなく、問題文の手がかり語から正解ラベルへ戻る処理を優先します。

## 検証した問い

- ITパスポートは、IT用語だけを覚える試験なのか
- 会社や社会の言葉の前提が分からないと、なぜ問題文が読みにくくなるのか
- 深く考える力が、試験中に逆に負荷になる場面はどこか
- 試験本番では、どこまで考えれば十分なのか

## 採用した見方

ITパスポートは、会社社会で使われる共通語を確認する試験として見ると処理しやすいです。

表向きは「ITの入門資格」ですが、実際には次のような言葉の前提も問われます。

- 会社の仕組み
- 組織の分け方
- 契約や知的財産
- お金、利益、費用
- 管理、評価、改善
- 業務を部品に分けて考える発想

このため、問題文が読みにくいときは、IT知識だけでなく「会社社会の抽象語彙」が不足している可能性があります。

## 深く考える領域

次のような話は、理解や納得のためには意味があります。

- 社会がなぜその言葉を重視するのか
- 会社がどのように人や業務を管理するのか
- お金、契約、評価、組織がどうつながるのか
- ある用語が、社会のどんな前提から生まれているのか

ただし、これは試験中に全部処理する領域ではありません。

深く考えるのは、復習や検証のときにします。

## 試験で処理する領域

試験中は、問題文が求めるラベルだけを拾います。

たとえば、ファンクションポイント法なら、社会や管理の背景まで考えすぎず、次の対応で切ります。

| 手がかり語 | 試験上のラベル |
| --- | --- |
| 入力画面、出力帳票、使用ファイル数 | ファンクションポイント法 |
| 似た過去システム | 類推法 |
| WBS、作業項目、積み上げ | 積算法 |
| プログラム行数、ステップ数 | LOC法・ステップ数見積り |

ここで必要なのは、世界を深く説明することではなく、選択肢を切るための対応表を思い出すことです。

## 認知アプローチ

ITパスポートは、深い理解を完全に捨てる試験ではありません。

ただし、本番で必要なのは「背景理解を使って、最終的には浅いラベル処理へ戻る力」です。

```text
背景を理解する
↓
問題文の手がかり語を見つける
↓
対応する試験ラベルへ戻す
↓
他の選択肢を切る
```

試験中に考察が広がりすぎたら、次の問いに戻します。

- この問題は何のカテゴリか
- 問題文の短い手がかり語はどれか
- 選択肢の中で、その手がかり語に対応するラベルはどれか
- 他の選択肢は、何のラベルなのか

## 残す判断

- ITパスポートは「ITだけ」ではなく、会社社会の共通語も問う試験として見る
- 背景理解は復習では有効だが、本番では問題文の手がかり語と選択肢のラベル対応を優先する
- 深く考える領域と、試験で処理する領域を分ける
- 本番では「社会の真実」ではなく、「この単語が出たらこの選択肢」という処理でよい
- 迷ったら、会社員用語カルタとして、問題文の札と選択肢の札を合わせる

## 試験側に戻さない内容

社会構造、既得権益、評価の相対性などの大きな考察は、試験の選択肢を切る直接材料にならない限り、`01_試験` 側には戻しません。

`01_試験` 側へ戻すのは、問題文の手がかり語、用語の住み分け、選択肢を切るための確認順だけにします。
