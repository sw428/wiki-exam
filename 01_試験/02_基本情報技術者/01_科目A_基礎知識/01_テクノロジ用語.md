# テクノロジ用語

## 目的

このファイルは、基本情報技術者試験の科目Aで出るテクノロジ系の基礎用語を、問題文の手がかりから選べる形に整理する場所です。

ITパスポートでは軽く分類するだけの用語でも、基本情報では少し踏み込んで、仕組みや前提まで確認します。

## カルノー図

### 何の問題か

カルノー図は、論理式を簡単にするための表です。

基本情報では、ITパスポートよりも、真理値表、論理式、論理回路とのつながりを意識します。

### 前提の流れ

カルノー図は単体で覚えるより、次の流れで見ます。

```text
0 と 1
-> AND / OR / NOT
-> 真理値表
-> 論理式
-> 論理式を簡単にする
-> カルノー図
```

### 試験上の意味

カルノー図は、真理値表や論理式の中で `1` になる条件をまとめ、論理式を短くするために使います。

問題で「論理式の簡単化」「論理回路の簡略化」のような言葉が出たら、カルノー図を疑います。

### 問題文の手がかり

- カルノー図
- 論理式
- 論理回路
- 真理値表
- AND / OR / NOT
- 1をまとめる
- 簡単化

### 似た用語との住み分け

| 用語 | 見るもの | 一瞬フレーズ |
| --- | --- | --- |
| 真理値表 | 入力の組合せと出力の一覧 | 0/1の全部表 |
| 論理式 | AND / OR / NOTで条件を表す式 | 条件を式で書く |
| 論理回路 | 論理演算を回路で表すもの | 式を回路にする |
| カルノー図 | 論理式を簡単にする表 | 1を囲んで短くする |

### 認知アプローチ

カルノー図が出たら、最初から作図を完璧にしようとせず、まず「何を簡単にしているのか」を見ます。

```text
真理値表や論理式
-> 1になる条件を見る
-> まとまりを探す
-> 論理式を短くする
```

基本情報では、この流れを理解しておくと、論理回路や論理式の問題に戻りやすくなります。

## 最後に見る一言
