# 情報セキュリティ整理マップ

## 目的

このフォルダは、基本情報技術者試験の情報セキュリティを整理する場所です。

攻撃、対策、認証、暗号、運用を、問題文の手がかり語から選べる形にします。

## 問題で見る手がかり

- 攻撃手法
- 脆弱性
- 認証、認可
- 暗号化、署名、証明書
- ログ、監視、運用
- インシデント対応

## まず切る選択肢

- 攻撃名と対策が対応していない選択肢
- 暗号化とデジタル署名を混同している選択肢
- 認証と認可を混同している選択肢
- 技術対策と運用対策を取り違えている選択肢

## 認知アプローチ

セキュリティ問題では、最初に「攻撃を聞いているのか、対策を聞いているのか」を固定します。

```text
攻撃 -> 何を狙うか
対策 -> 何を防ぐか
認証 -> 誰か確認
認可 -> 何を許すか
暗号化 -> 隠す
署名 -> 改ざん検知と本人性
```
