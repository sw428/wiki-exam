# Javaゴールド整理マップ

## 目的

このフォルダは、Java Gold の高度な文法、API、並行処理、I/O、Stream、ラムダなどの安定知識を置く場所です。

Java Gold では、API名の暗記だけではなく、戻り値、評価順、例外発生位置、型推論、スレッド安全性を見分けることを重視します。

## 置くもの

- Java Gold の出題範囲に入る概念整理
- Stream / ラムダ / ジェネリクス / 並行処理の判断基準
- API問題を読むときの確認順
- Silver知識との違い
- 似たAPIや似た型指定の住み分け
- [`02_検証/05_Javaゴールド`](../../02_検証/05_Javaゴールド/00_Javaゴールド検証マップ.md) から戻した安定知識

## 問題での基本判断

```text
APIの型 -> 戻り値 -> 遅延評価か即時実行か -> 例外位置 -> 副作用やスレッド安全性
```

Goldでは、「知っているAPI」に見えるものほど誤答しやすいので、似たAPIとの差を先に見ます。

API名の意味だけで判断せず、代入先の型、戻り値、評価が起きる位置、例外、副作用をコード上で追います。必要な観点だけを選び、すべてのコード問題へ同じ説明見出しを強制しません。

## 処理の流れ

1. コード問題や誤答は、まず検証側に置く
2. 間違えた理由を「API理解 / 実行順 / 型推論 / 例外 / 並行処理 / Stream操作」に分ける
3. 次回も使う判断基準だけを、このフォルダへ戻す
4. 分野が決まる内容は各カテゴリへ、横断する読み順はこの `00_` に置く

## 置かないもの

- 解きかけのコード問題ログ
- API名だけの暗記メモ
- Java Silver側で十分な基礎文法整理

## カテゴリ追加ルール

- 必要なカテゴリは、この `05_Javaゴールド` フォルダ配下に作る
- カテゴリやファイルを追加するときは、`01_...` / `02_...` の連番をこのフォルダ内で合わせる
- 知識整理では、APIカテゴリ、評価順、型推論、例外、並行処理、似たAPIや操作の住み分けを優先する
