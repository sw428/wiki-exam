# 並行処理整理マップ

## 目的

Java Gold の並行処理を、スレッド生成、Executor、同期、競合、スレッド安全性で整理します。

## 問題で見る手がかり

- `Thread` / `Runnable` / `Callable`
- `ExecutorService`
- `Future`
- `synchronized`
- race condition
- thread-safe collection

## まず切る選択肢

- `run()` と `start()` を混同している選択肢
- `Runnable` と `Callable` の戻り値や例外を混同している選択肢
- 同期されていない共有状態を安全だとしている選択肢
- 実行順が常に固定されると思っている選択肢

## 似たAPIとの住み分け

- `Runnable` は戻り値なし
- `Callable` は戻り値あり、例外も扱いやすい
- `Future` は非同期処理の結果を表す
- `synchronized` は排他制御

## 認知アプローチ

並行処理では、出力順を決め打ちしません。共有データ、同期の有無、結果取得のタイミングを先に見ます。
