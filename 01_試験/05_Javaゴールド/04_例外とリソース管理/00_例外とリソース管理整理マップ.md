# 例外とリソース管理整理マップ

## 目的

Java Gold の例外処理、try-with-resources、抑制例外、AutoCloseable を整理します。

## 問題で見る手がかり

- try-with-resources
- `AutoCloseable`
- `close()` の呼び出し順
- suppressed exception
- multi-catch
- 例外の優先順位

## まず切る選択肢

- リソースのクローズ順を取り違えている選択肢
- 本体例外と `close()` 例外の優先順位を混同している選択肢
- multi-catch で同じ継承関係の例外を並べているコード
- 抑制例外を主例外として扱っている選択肢

## 似た構文との住み分け

- try-with-resources はリソースを逆順に閉じる
- 本体で例外が出て、closeでも例外が出ると、close側は抑制例外になりやすい
- `AutoCloseable` は `close()` を持つ
- multi-catch では、親子関係のある例外を同時に並べられない

## 認知アプローチ

例外とリソース管理では、例外の発生位置とクローズ順を別々に追います。
