# ジェネリクス整理マップ

## 目的

ジェネリクスを、型パラメータ、ワイルドカード、上限・下限境界で判断できる形に整理します。

## 問題で見る手がかり

- `<T>`
- `? extends ...`
- `? super ...`
- 型推論
- コレクションへの追加可否
- raw type

## まず切る選択肢

- `extends` と `super` の向きを取り違えている選択肢
- 読める型と追加できる型を混同している選択肢
- raw type による警告や実行時リスクを見落としている選択肢

## 似た構文との住み分け

- `? extends T` は T以下を読む側に寄る
- `? super T` は T以上へ入れる側に寄る
- 具体的な型パラメータは、その型として扱う
- raw type は型安全性が弱くなる

## 認知アプローチ

ジェネリクス問題では、最初に「読むのか、入れるのか」を見ます。次に、境界が上限か下限かを確認します。
