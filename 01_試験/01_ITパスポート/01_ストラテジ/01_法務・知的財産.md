# 法務・知的財産

## 目的

このファイルは、ITパスポートの法務・知的財産系の用語を、問題で選択肢を切れる形に整理する場所です。

## ざっくり分類

| 手がかり語 | 用語 | 住み分け |
| --- | --- | --- |
| 発明 / 技術的アイデア | 特許 | 技術的な発明を守る |
| ロゴ / マーク / ブランド名 | 商標 | 商品やサービスの目印を守る |
| 文章 / 取扱説明書 / 音楽 / 絵 / プログラム表現 | 著作権 | 表現そのものを守る |
| 通信販売 / 訪問販売 / 売り方 | 特定商取引法 | 消費者を守るため、売り方を規制する |
| ずるい商売 / まぎらわしい表示 / 営業秘密 | 不正競争防止法 | 公正な競争を妨げる行為を防ぐ |
| 氏名 / 住所 / 電話番号 / 名刺 / 社員名 | 個人情報 | 特定の個人が分かる |
| 特許 / 相互に許諾 / お互いに使う | クロスライセンス | 特許や技術を使う許可を出し合う |
| 包装 / 開封 / 使用許諾条件 | シュリンクラップ契約 | 開封した時点で同意したとみなす |

## 判別ポイント

- 発明なら特許
- ロゴなら商標
- 表現なら著作権
- 売り方なら特定商取引法
- ずるい商売なら不正競争防止法
- 特許を相互に使わせるならクロスライセンス

## 似た用語との住み分け

- 特許は、ロゴやブランド名を守るものではない
- 商標は、技術的な発明を守るものではない
- 著作権は、アイデアそのものではなく表現を守る
- 取扱説明書は、文章として書かれた表現物なので著作権で見る
- 特定商取引法は、権利を守る法律というより、販売方法を規制する法律
- 不正競争防止法は、他社とまぎらわしい表示や営業秘密など、公正な競争を妨げる行為を扱う
- 記号や数字だけの匿名ハンドルネームは、それだけでは個人情報ではない寄りで見る
- 名刺、電話帳、会社名と役職と社員名の組合せは、個人が分かるので個人情報として見る

## シュリンクラップ契約の見分け方

### 何の問題か

シュリンクラップ契約は、ソフトウェアなどの包装を開封した時点で、使用許諾条件に同意したとみなす契約です。

問題文では、箱や包装を開けた時点で契約が成立するかを見ます。

### 小学生レベルの言い換え

「箱を開けたら、このソフトのルールにOKしたことにします」という約束です。

昔のゲームソフトやパソコンソフトの箱で考えると分かりやすいです。

```text
包装を開ける
↓
使用許諾条件に同意したとみなす
```

### 試験上の意味

ITパスポートでは、シュリンクラップ契約を「開封した時点で契約成立」として見ます。

使用許諾条件を細かく読んでいなかったとしても、包装を開封したことが同意の意思表示として扱われる、という問題です。

### 問題文の手がかり

- シュリンクラップ契約
- 包装
- 開封
- 使用許諾条件
- 契約成立

### 選択肢の切り方

- 包装を開封すると、使用許諾条件を理解していなかったとしても契約が成立するなら、シュリンクラップ契約
- 開封しても一定期間なら無効にできる、という話ではない
- 開封しても使い始めなければ無効、という話ではない
- 開封していないのに一定期間で契約成立、という話ではない

### 誤答しやすい理由

「使い始めていないなら契約していないのでは」と考えると迷いやすいです。

シュリンクラップ契約で見るポイントは、使用開始ではなく開封です。

### 最後に見る一言

シュリンクラップ契約 = 箱を開けたら使用許諾に同意。

## 著作権の見分け方

### 何の問題か

著作権の問題では、文章、図、画像、音楽、プログラムなど、「作った表現」を守る権利かどうかを見ます。

ソフトウェアに付いている取扱説明書は、操作方法というアイデアそのものではなく、文章として書かれた表現物なので著作権で守られます。

### 問題文の手がかり

- 取扱説明書
- 文章
- 図
- 画像
- 音楽
- プログラム
- 表現

### 選択肢の切り方

- 取扱説明書や文章なら、著作権
- 商品の形や見た目のデザインなら、意匠権
- 商品名、ブランド名、ロゴ、マークなら、商標権
- 新しい技術、発明、仕組み、方法なら、特許権

### 誤答しやすい理由

ソフトウェアに付いている説明書だと、つい「ソフトウェアの仕組み」や「技術」の話に見えます。

でも問題文が聞いているのは、ソフトウェアの発明ではなく、説明書として書かれた文章や内容を守る権利です。だから特許権ではなく著作権で見ます。

### 最後に見る一言

説明書 = 文章 = 著作権。

## クロスライセンスの見分け方

### 何の問題か

クロスライセンスは、会社同士が互いの特許や技術を使うことを許可し合う契約です。

問題文では、`相互に許諾` が強い手がかりになります。

### 英単語としての覚え方

これは大学受験の一般英単語帳で覚える英単語というより、ビジネス、法律、IT、知的財産のカタカナ専門語として覚える用語です。

`cross` は「交差する」「互いに行き来する」イメージで使います。

```text
会社Aの特許を会社Bが使う
会社Bの特許を会社Aが使う
```

このように、許可が互いに交差するのでクロスライセンスです。

### 小学生レベルの言い換え

会社Aと会社Bが、次のように言い合う状態です。

```text
A: 君の特許を使わせてね
B: いいよ。その代わり、こっちの特許も使っていいよ
```

一方だけが使わせてもらうのではなく、お互いに使わせ合うのがポイントです。

### 似た用語との住み分け

| 用語 | 見るもの | 一瞬フレーズ |
| --- | --- | --- |
| アライアンス | 企業同士の協力関係 | 協力しよう |
| クロスライセンス | 特許や技術を相互に許諾 | 特許を使わせ合う |
| ジョイントベンチャー | 共同で新会社や事業を作る | 一緒に作る |
| ロイヤルティ | 特許や著作物の使用料 | 使う代わりに払うお金 |

### 選択肢の切り方

- `相互に許諾`、`互いの特許を使う` ならクロスライセンス
- 企業同士の広い協力関係ならアライアンス
- 共同で新しい会社や事業を作るならジョイントベンチャー
- 使わせてもらう対価として支払うお金ならロイヤルティ

### 誤答しやすい理由

アライアンスもジョイントベンチャーも、会社同士が協力しているように見えます。

ただし、問題文が聞いている中心が「特許をお互いに使わせる契約」なら、広い協力関係ではなくクロスライセンスです。

### 最後に見る一言

特許を相互に許諾 = クロスライセンス。

## 個人情報保護法の見分け方

### 何の問題か

個人情報保護法の問題では、「特定の個人が分かる情報か」を見ます。

名前だけでなく、会社名、役職、住所、電話番号、メールアドレスなどと組み合わせて、誰のことか分かる場合も個人情報として見ます。

### 問題文の手がかり

- 氏名
- 住所
- 電話番号
- 社員名
- 役職
- 名刺
- 電話帳
- 担当者データベース

### 選択肢の切り方

- 社員名があるなら、個人情報として見る
- 電話帳の氏名、住所、電話番号は、個人情報として見る
- 名刺データベースは、担当者個人が分かるので個人情報として見る
- 記号や数字だけのハンドルネームだけなら、特定の個人が分からないため個人情報ではない寄りで見る

### 誤答しやすい理由

「会社の情報だから個人情報ではない」と考えると間違えやすいです。

会社名や役職だけなら会社情報に見えますが、そこに社員名や担当者名が入ると、特定の個人が分かる情報になります。

### 最後に見る一言

名前、名刺、電話帳が出たら個人情報。記号だけの匿名IDは個人情報ではない寄り。

## 一瞬フレーズ

```text
発明 = 特許
ロゴ = 商標
説明書・文章 = 著作権
表現 = 著作権
売り方 = 特定商取引法
ずるい商売 = 不正競争防止法
名前・名刺・電話帳 = 個人情報
特許を使わせ合う = クロスライセンス
開封したら同意 = シュリンクラップ契約
```
