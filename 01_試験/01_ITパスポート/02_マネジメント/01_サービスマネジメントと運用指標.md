# サービスマネジメントと運用指標

## 目的

このファイルは、ITパスポートで混同しやすいサービス運用指標を、問題で切れる形に整理する場所です。

## ざっくり分類

| 用語 | 見るもの | 一瞬フレーズ |
| --- | --- | --- |
| MTBF | 故障から次の故障までの平均時間 | 壊れにくさ |
| MTTR | 修理にかかる平均時間 | 直る速さ |
| MTBSI | サービス停止から次の停止までの平均時間 | 止まる間隔 |
| 稼働率 | 使える時間の割合 | どれだけ動いているか |

## 判別ポイント

- `MTBF` は、故障と故障の間を見る
- `MTTR` は、故障してから復旧するまでを見る
- `MTBSI` は、サービス停止と次の停止の間を見る
- 稼働率は、稼働時間と停止時間の比率で見る

## 似た用語との住み分け

- `MTTR` は直る時間で、止まる間隔ではない
- `MTBSI` は止まる間隔で、修理時間ではない
- `MTBF` は機器やシステムの故障間隔を見やすい
- `MTBSI` はサービス停止を単位にするため、利用者目線の停止間隔に寄る

## 一瞬フレーズ

```text
BF = Between Failure = 故障と故障の間
TR = To Repair = 直すまで
BSI = Between Service Incidents = サービス停止の間
```

## FAQ化と一次回答率

### 何の問題か

過去のインシデントをFAQ化する問題では、サービスデスクが利用者からの問い合わせに早く答えられるようになるかを見ます。

FAQは、よくある問い合わせと回答をまとめた知識集です。

### 問題文の手がかり

- 過去のインシデント
- FAQ
- よくある問い合わせ
- サービスデスク
- 一次回答率

### 試験上の意味

過去のトラブル内容と解決方法がFAQにあると、同じような問い合わせが来たときに、担当者がすぐ回答しやすくなります。

そのため、利用者からの問い合わせに対する一次回答率が高まります。

### 似た用語との住み分け

| 見るもの | 用語・考え方 | 切り方 |
| --- | --- | --- |
| よくある問い合わせへすぐ答える | FAQ | 一次回答率アップ |
| 構成要素の情報を確認する | CMDB / 構成管理DB | FAQではない |
| 要員候補の経歴を確認する | 人材管理 / スキル管理 | FAQではない |

### 最後に見る一言

FAQ化 = 問い合わせ対応が速くなる。FAQ = 一次回答率アップ。

## 可用性管理

### 一言でいうと

必要なときに、ITサービスを使える状態にしておく管理です。

### 問題文の手がかり

- 必要なときに提供できる
- 利用できる
- 稼働している
- 停止時間

### 似た用語との住み分け

| 用語 | 見るもの | 切り方 |
| --- | --- | --- |
| 可用性管理 | 必要なときにサービスを使えるか | 今、使える状態 |
| ITサービス継続性管理 | 災害や大事故の後も継続・復旧できるか | 大きな事故から戻す |
| キャパシティ管理 | 必要な処理能力が足りるか | 能力・容量 |
| サービスレベル管理 | 合意したサービス水準を守れているか | 約束した水準 |

### 選択肢の切り方

問題文の中心が「必要なときに提供できる」なら、復旧計画や処理能力ではなく、可用性管理を選びます。

今回の問題の正解は可用性管理です。
