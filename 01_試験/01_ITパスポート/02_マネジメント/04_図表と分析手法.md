# 図表と分析手法

## 何の問題か

このファイルは、ITパスポートで出る図表や分析手法を、問題文の手がかりから選べるように整理する場所です。

図の名前だけを覚えるより、「何個の数値を見るのか」「関係を見るのか」「原因を見るのか」「順番を見るのか」を先に見ます。

## 一瞬フレーズ

| 手がかり | 図表 | 見るもの |
| --- | --- | --- |
| 2つの数値の関係 | 散布図 | 相関、関係 |
| 3つの数値を丸の大きさまで使って見る | バブルチャート | 横、縦、丸の大きさ |
| 時間順に安定しているか見る | 管理図 | 上限、下限、ばらつき |
| 原因を骨のように整理する | 特性要因図 | 原因と結果 |
| 多い順、重要順に並べる | パレート図 | 重要な要因 |
| 作業日程を横棒で見る | ガントチャート | スケジュール |
| 考えを枝分かれで整理する | マインドマップ | アイデア |
| 将来計画や段階を並べる | ロードマップ | 予定、道筋 |

## 散布図

散布図は、2つの数値の関係を見る図です。

例:

```text
横: プログラムの行数
縦: 不具合件数
```

点をばらまいて、行数が多いほど不具合も増えそうかを見るときに使います。

## バブルチャート

バブルチャートは、3つの数値を同時に見る図です。

例:

```text
横: 座席数
縦: 客単価
丸の大きさ: 売上高
```

散布図に、丸の大きさという3つ目の情報を足したものとして見ると切りやすいです。

## 管理図

管理図は、品質や作業結果が普通の範囲に収まっているかを、上限・下限の管理限界線で見る図です。

例:

```text
横: プログラムや工程
縦: 単位ステップ数当たりのバグ数
線: 上限・下限の管理限界
```

上限を超えたら「異常にバグが多い」と見ます。

下限を下回った場合も、「本当に少ないのか」「テストが甘くて見つけられていないのか」を疑うので、異常として見ます。

## 特性要因図

特性要因図は、結果に対する原因を魚の骨のように整理する図です。

「なぜバグが多いのか」「なぜ納期が遅れたのか」のように、原因を分解して探すときに使います。

## パレート図

パレート図は、件数や影響の大きいものから順に並べ、重要な要因を見つける図です。

「どの種類の不具合が多いか」「どの原因から対策すべきか」を見るときに使います。

## レーダーチャート

レーダーチャートは、複数の項目のバランスを見る図です。

能力値、評価項目、複数観点の強弱を、中心から外側へ広がる形で見ます。

## 選択肢の切り方

- 2つの数値の関係、相関を見るなら `散布図`
- 3つの数値を、横、縦、丸の大きさで見るなら `バブルチャート`
- 上限・下限の限界線を超えた異常を見つけるなら `管理図`
- 原因を魚の骨のように整理するなら `特性要因図`
- 多い順や重要順に並べるなら `パレート図`
- 複数項目のバランスを見るなら `レーダーチャート`
- 作業の予定や期間を見るなら `ガントチャート`
- 発想や考えを広げるなら `マインドマップ`
- 将来計画や段階を示すなら `ロードマップ`

## 誤答しやすい理由

選択肢が全部「何かを整理する図」に見えると混ざります。

先に見るのは図の名前ではなく、問題文の数値の数です。

- 2つの数値なら散布図
- 3つ目を丸の大きさで表すならバブルチャート
- 上限・下限で異常検出なら管理図
- 原因の整理なら特性要因図
- 多い順で重要度を見るならパレート図
- バランスを見るならレーダーチャート

## 最後に見る一言

上限・下限なら管理図。原因なら特性要因図。多い順ならパレート図。バランスならレーダーチャート。
