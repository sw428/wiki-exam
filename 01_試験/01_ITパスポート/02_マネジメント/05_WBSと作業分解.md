# WBSと作業分解

## 何の問題か

WBSは、プロジェクトの大きな作業を階層で分解して表す手法です。

問題文に、作業を細かく分けた木のような図が出たらWBSを疑います。

## 読み方

`WBS` は「ダブリュービーエス」と読みます。

`Work Breakdown Structure` の略です。

```text
Work = 作業
Breakdown = 分解
Structure = 構造
```

つまり、作業を分解した構造です。

## 問題文の手がかり

- プロジェクト作業
- 階層
- 分解
- 作業項目
- 大きい仕事を小さい仕事に分ける

## 小学生レベルの言い換え

WBSは、大きな宿題を小さな作業に分けた表です。

```text
新営業店システム開発プロジェクト
├ 要件定義
├ システム設計
├ プログラミング
└ 結合テスト
   ├ テスト計画
   └ テスト実施
```

大きい仕事をそのまま見ると何をすればよいか分かりにくいので、小さい作業へ分けます。

## 似た用語との住み分け

| 用語 | 見るもの | 一瞬フレーズ |
| --- | --- | --- |
| WBS | 作業を階層で分解 | 作業分解の木 |
| PERT | 作業の順番、日程、依存関係 | 矢印で日程 |
| アローダイアグラム | 作業順と所要時間 | 矢印で順番 |
| ファンクションポイント法 | 機能量から規模見積り | 利用者から見える機能 |
| 構造化プログラミング | プログラムの制御構造 | 順次、分岐、繰返し |

## 選択肢の切り方

- 階層で作業分解なら `WBS`
- 矢印で作業順や日程を見るなら `PERT` やアローダイアグラム
- 機能の数から開発規模を見積もるならファンクションポイント法
- プログラムの構造を聞いているなら構造化プログラミング

## 最後に見る一言

階層で作業分解 = WBS。矢印で日程 = PERT。
