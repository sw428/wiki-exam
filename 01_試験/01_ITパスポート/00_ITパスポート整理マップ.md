# ITパスポート整理マップ

## 目的

このフォルダは、ITパスポートの安定知識と確認順を置く場所です。

この `00_` ファイルは、用語カテゴリ、3分類の入口、試験本番で選択肢を切るための判別軸をまとめる入口として扱います。

## 置くもの

- ITパスポートの重要語句
- ストラテジ / マネジメント / テクノロジの基本整理
- 問題文の読み方
- 選択肢の消し方
- 繰り返し間違える用語の区別
- カテゴリ別の知識整理
- 似た用語との住み分け
- 試験で選択肢を切るための認知アプローチ
- [`02_検証/01_ITパスポート`](../../02_検証/01_ITパスポート/00_ITパスポート検証マップ.md) から戻した安定知識

## 問題単位の基本判断

ITパスポートでは、まず問題がどの大分類に属するかを見ます。

```text
会社・経営・法律・業務・システム戦略 -> ストラテジ
プロジェクト・サービス運用・障害対応・開発管理 -> マネジメント
通信・コンピュータ・セキュリティ・データベース -> テクノロジ
```

分類が決まると、関係ない選択肢を先に切りやすくなります。

## 3分類の入口

- [01_ストラテジ](./01_ストラテジ/00_ストラテジ整理マップ.md)
  - 会社、経営、法務、知的財産、業務、システム戦略
- [02_マネジメント](./02_マネジメント/00_マネジメント整理マップ.md)
  - プロジェクト、サービス運用、障害対応、開発管理
- [03_テクノロジ](./03_テクノロジ/00_テクノロジ整理マップ.md)
  - 通信、コンピュータ、セキュリティ、データベース、基礎理論

## 試験での認知アプローチ

ITパスポートの整理では、短縮ではなく認知変換を優先します。

問題ログやGPT回答は、ただ短くするのではなく、次に同じタイプの問題を見たときに、問題文の手がかりから正解へ戻れる形へ並べ替えます。

### 1. まずジャンルを見る

問題文を読んだら、最初に「ストラテジか、マネジメントか、テクノロジか」を見ます。

### 2. 次に手がかり語を見る

用語の正式名称より、問題文の短い手がかり語を優先します。

- 声 = `VoIP`
- 時刻 = `NTP`
- かざす = `NFC`
- 格安SIM = `MVNO`
- 中身を隠す = 暗号化
- 改ざん検知 = デジタル署名
- ロゴ = 商標
- 発明 = 特許

### 3. 最後に似た用語との差を見る

- デジタル署名は、中身を隠すものではない
- 暗号化は、改ざん検知そのものではない
- 商標は、発明を守るものではない
- 特許は、ロゴを守るものではない
- `MTTR` は直る時間で、止まる間隔ではない
- `MTBSI` は止まる間隔で、直る時間ではない

### 4. 短い暗記の前に意味の橋を置く

一瞬フレーズは残しますが、その前に次のどれかを置きます。

- 小学生レベルの言い換え
- 普通の言葉と試験上の意味のズレ
- 日常例や具体物
- 選択肢を切る理由
- 誤答しやすい読み違い

## 置かないもの

- 解きかけの過去問ログ
- 正誤だけのメモ
- GPT回答を確認していない未整理メモ
- 学習方法や記憶方法そのものの検証ログ

## 処理の流れ

1. 問題演習や誤答は、まず検証側に置く
2. 間違えた理由を「用語不足 / 読み違い / 選択肢比較ミス / 暗記不足」に分ける
3. 次回も使う確認順だけを、このフォルダへ戻す
4. 3分類をまたぐ判別軸はこの `00_` に、分野内の知識は各カテゴリへ置く

## カテゴリ追加ルール

- 必要なカテゴリは、この `01_ITパスポート` フォルダ配下に作る
- カテゴリやファイルを追加するときは、`01_...` / `02_...` の連番をこのフォルダ内で合わせる
- 追加したカテゴリは、この `00_` マップの「3分類の入口」または「今後の候補」に反映する
- 知識整理では、用語数を増やすより、選択肢で迷わない住み分けを優先する
