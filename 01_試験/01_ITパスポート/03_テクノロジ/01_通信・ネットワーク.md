# 通信・ネットワーク

## 目的

このファイルは、ITパスポートの通信・ネットワーク系用語を、短い手がかり語で選択肢を切れる形に整理する場所です。

## 一瞬フレーズ

| 手がかり語 | 用語 | 見るところ |
| --- | --- | --- |
| 音声 / インターネット電話 | VoIP | IPネットワークで音声通話 |
| 時刻合わせ | NTP | コンピュータの時刻同期 |
| かざす / 近距離無線 | NFC | 近距離で通信 |
| 格安SIM / 回線を借りる | MVNO | 通信設備を借りてサービス提供 |
| 名前からIPアドレス | DNS | ドメイン名をIPアドレスへ対応させる |
| IPアドレスを自動で配る | DHCP | LAN内のPCへIPアドレスを自動割当 |
| 現場近くで先に処理 / IoTゲートウェイ | エッジコンピューティング | 遠くへ送る前に近くで処理 |

## 判別ポイント

- 声が出たら `VoIP`
- 時刻なら `NTP`
- かざすなら `NFC`
- 格安SIMや回線を借りる事業者なら `MVNO`
- 名前解決なら `DNS`
- LANにつないだPCがIPアドレスを自動でもらうなら `DHCP`
- カメラ、センサ、IoTゲートウェイなどの近くで先に処理するなら `エッジコンピューティング`

## 似た用語との住み分け

- `NFC` は近距離通信であり、インターネット電話ではない
- `NTP` は時刻同期であり、暗号化ではない
- `DNS` は名前解決であり、通信の暗号化そのものではない
- `MVNO` は通信事業者の形であり、通信方式そのものではない
- `DHCP` はIPアドレスを配る仕組みであり、ウイルス定義ファイル、迷惑メールリスト、メールアドレスを配るものではない
- エッジコンピューティングは、端末や現場の近くで処理することであり、中央サーバやクラウドで全部処理する話ではない

## エッジコンピューティング

### 何の問題か

エッジコンピューティングは、中央のサーバやクラウドへ全部送る前に、端末や現場に近い場所で先に処理する考え方です。

問題文では、店舗のカメラ映像やセンサ情報を、IoTゲートウェイなどで先に分析してから送る形で出やすいです。

### 小学生レベルの言い換え

遠くの大きなコンピュータへ全部送る前に、近くの小さいコンピュータで先に考える仕組みです。

普通のやり方:

```text
防犯カメラ
↓
動画を全部送る
↓
遠くのサーバで分析
```

エッジコンピューティング:

```text
店舗の中のIoTゲートウェイ
↓
必要な部分だけ先に分析
↓
軽くして送る
```

### 試験上の意味

`edge` は「端」「はしっこ」という意味で見ます。

試験では、ネットワークの中心ではなく、利用者、センサ、カメラ、店舗、工場などに近い場所で処理することをエッジコンピューティングとして見ます。

### 問題文の手がかり

- エッジ
- IoT
- IoTゲートウェイ
- 店舗や工場などの現場
- カメラ映像やセンサデータ
- データを送る前に処理
- 通信量を減らす
- 応答を速くする

### 何を減らしたい技術か

エッジコンピューティングは、主に次の負荷を減らすために使います。

- 送るデータ量
- 通信回線の負荷
- 遠くのサーバへ送って戻るまでの遅れ
- クラウド側で全部処理する負担

### 似た用語との住み分け

| 用語 | 見るもの | 一瞬フレーズ |
| --- | --- | --- |
| MDM | スマホやタブレットなどの端末管理 | 端末を管理する |
| SDN | ネットワークをソフトウェアで制御 | ネットワーク制御 |
| エッジコンピューティング | 現場近くで先に処理 | 近くで考える |
| デュプレックスシステム | コンピュータを二重化して可用性を高める | 壊れても動くように二重化 |

### 選択肢の切り方

- `IoTゲートウェイ`、`映像分析`、`送信前に処理`、`現場近くで処理` なら、エッジコンピューティング
- スマホやタブレットを一元管理するなら、MDM
- ネットワークをソフトウェアで制御するなら、SDN
- コンピュータを二重化して障害に備えるなら、デュプレックスシステム

### 最後に見る一言

エッジコンピューティング = 遠くに全部送る前に、近くで先に処理する。
