# セキュリティ

## 目的

このファイルは、ITパスポートのセキュリティ用語を、問題文の手がかり語と選択肢の切り方で整理する場所です。

## 一瞬フレーズ

| 手がかり語 | 用語 | 見るところ |
| --- | --- | --- |
| 中身を見られないようにする | 暗号化 | 秘密にする |
| 改ざんされていないことを確認 | デジタル署名 | 改ざん検知と本人性 |
| 本人か確認する | 認証 | 利用者を確かめる |
| なりすましを防ぐ | 多要素認証 | 複数の要素で確認 |
| 悪意あるソフト | マルウェア | ウイルスなどの総称 |
| 不正アクセス / 予兆 / 検知 / 通知 | IDS | 見つけて管理者に知らせる |
| 事前共有鍵 / Wi-Fiの合言葉 | PSK | 暗号化鍵を作る元 |
| 誰が何をできるか / 権限 | アクセス管理 | 利用できる範囲の制御 |
| ログイン記録 / ID / 日時 / 監査 | アクセス記録の監査 | 利用権限の運用確認 |

## 判別ポイント

- 隠すなら暗号化
- 改ざん検知ならデジタル署名
- 本人確認なら認証
- 複数要素なら多要素認証
- 悪意あるソフト全体ならマルウェア
- 不正アクセスを検知して通知するなら `IDS`
- `WPA2`、`PSK`、`パスフレーズ`、`事前共有鍵` が出たら、Wi-Fi接続時の合言葉を見る
- 誰がどの情報を見たり編集したりできるかなら、アクセス管理を見る
- ログイン記録をためて後から監査するなら、システム利用権限が正しく運用されているかを見る

## 似た用語との住み分け

- 暗号化は、中身を隠すことが中心で、改ざん検知そのものではない
- デジタル署名は、中身を隠すより、改ざん検知と署名者確認に寄る
- 認証は本人確認であり、通信内容を秘密にすることではない
- ウイルスはマルウェアの一種であり、マルウェアはより広い総称
- `PSK` はWi-Fiの合言葉であり、Wi-Fiの名前である `SSID` ではない
- `DHCP` はIPアドレスを配る仕組みであり、PSKそのものではない

## アクセス管理の見分け方

### 何の問題か

アクセス管理は、システムやデータに対して、誰が何をできるかを制御する話です。

見るだけでよい人、編集できる人、使ってはいけない人、部署や役割ごとの権限を分けるときに出ます。

### 小学生レベルの言い換え

アクセス管理は、部屋の鍵と入ってよい人の名簿です。

誰でも全部の部屋に入れるのではなく、「この人はここまで」「この人は入れない」と決めます。

### 問題文の手がかり

- アクセス管理
- アクセス権限
- 利用権限
- 誰が見てよいか
- 誰が編集してよいか
- 業務分掌

### 開発工程との関係

アクセス管理そのものはセキュリティの話です。

ただし、`検討を開始するプロセス` を聞かれたら、開発工程の問題として見ます。誰が何をできるかは業務ルールに関係するので、最初の要件定義から決めます。

詳しくは [システム開発管理のアクセス管理と要件定義](../02_マネジメント/06_システム開発管理.md#アクセス管理と要件定義) に戻ります。

### 最後に見る一言

アクセス管理 = 誰が何をできるか。検討開始なら要件定義。

## WPA2のPSKの見分け方

### 何の問題か

WPA2のPSKは、無線LANのセキュリティ用語問題です。

問題文では、アクセスポイントへ接続するときの認証や、通信を暗号化するための鍵が生成される元として出ます。

### 小学生レベルの言い換え

PSKは、Wi-Fiに入るための合言葉です。

家のWi-Fiに入るとき、最初に「この合言葉を知っている人だけ入ってよい」と確認します。その合言葉がPSKです。

### 試験上の意味

`PSK` は `Pre-Shared Key` の略で、事前共有鍵です。

```text
Pre = 前もって
Shared = 共有された
Key = 鍵
```

WPA2では、PSKを元にして通信の暗号化に使う鍵が作られます。

### 選択肢の切り方

- `パスフレーズ`、`事前共有鍵`、`暗号化に用いる鍵が生成` なら `PSK`
- アクセスポイントを識別する名前なら `SSID`
- IPアドレスを自動で割り当てる仕組みなら `DHCP`
- ネットワーク上の住所そのものなら `IPアドレス`

### 誤答しやすい理由

Wi-Fiの問題では、`PSK`、`SSID`、`IPアドレス`、`DHCP` が並んで出やすいです。

全部「Wi-Fiにつなぐときに関係ありそう」に見えますが、役割が違います。

| 用語 | 役割 |
| --- | --- |
| PSK | Wi-Fiの合言葉、暗号化鍵を作る元 |
| SSID | Wi-Fiの名前 |
| IPアドレス | ネットワーク上の住所 |
| DHCP | IPアドレスを配る仕組み |

### 最後に見る一言

PSKはWi-Fiの名前ではなく、暗号化の元になる合言葉。

## ログイン記録の監査の見分け方

### 何の問題か

ログイン記録の監査は、アクセス管理の問題です。

問題文では、ログインした記録をためて、あとで何を確認するかが聞かれます。

### 小学生レベルの言い換え

ログイン記録は、教室の入室カードの記録です。

誰が、いつ、入れたか、入れなかったかを後で見て、「入ってよい人だけが入っているか」を確認します。

### 試験上の意味

ログイン記録には、利用者ID、ログイン日時、成功・失敗、アクセス先などが残ります。

これを監査すると、システム利用権限が正しく運用されているかを確認できます。

### 選択肢の切り方

- `ログイン記録`、`ID`、`日時`、`認証結果`、`監査` なら、利用権限の運用確認
- 障害、エラー、停止、復旧だけを聞いているなら障害管理
- 使いやすさ、操作性、満足度なら利便性やユーザビリティ
- 誰が入れるか、入ってよいかを見ているなら権限管理

### 誤答しやすい理由

ログイン失敗の記録があると、障害情報に見えることがあります。

ただし、問題文の中心が「誰が、いつ、ログインできたかを監査する」なら、見る対象は障害ではなく利用権限です。

### 最後に見る一言

ログイン記録の監査 = システム利用権限が正しく運用されているかの確認。

## IDSの見分け方

### 何の問題か

IDSは、ネットワークセキュリティの用語問題です。

ITパスポートでは、主に「ネットワーク環境で利用されるIDSの役割」を聞かれます。

### 小学生レベルの言い換え

IDSは、怪しいアクセスを見つけて管理者に知らせる防犯カメラです。

泥棒を直接止めるというより、「怪しい動きがあります」と知らせる役です。

### 試験上の意味

IDSは `Intrusion Detection System` の略で、侵入検知システムです。

試験では、次の形で覚えます。

```text
不正アクセスやその予兆を検知し、管理者に通知する
```

強い手がかりは `検知` と `通知` です。

### 似た用語との住み分け

| 用語 | 一瞬フレーズ | 役割 |
| --- | --- | --- |
| IDS | 見つけて知らせる | 不正アクセスを検知して通知 |
| IPS | 見つけて止める | 不正アクセスを検知して防御 |
| ファイアウォール | 通信の門番 | 許可された通信だけ通す |
| WAF | Webアプリの盾 | Webアプリへの攻撃を防ぐ |

### 選択肢の切り方

- `IPアドレス` / `ドメイン名` / `変換` は `DNS`
- `時刻` / `同期` は `NTP`
- `不正アクセス` / `予兆` / `検知` / `通知` は `IDS`
- `メールサーバ` / `メールクライアント` / `電子メール` はメール受信系

### フロントエンド・実務との関係

IDSは、フロントエンドで直接実装するものではありません。

ただし、画面からAPIへ通信するとき、その裏側にはサーバ、ネットワーク、ログ、IDS、WAF、ファイアウォールなどの防御や監視が関係します。

フロントエンド学習で優先度が高いのは、XSS、Cookie、認証、認可、HTTPS、CORS、APIキー、フォーム入力の扱いです。IDSは、その周辺構造を理解するための補助知識として扱います。

### 最後に見る一言

IDS = 不正アクセスを見つけて知らせる防犯カメラ。
