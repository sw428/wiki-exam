# 繰返し処理と合計計算

## 何の問題か

これは、プログラムの繰返し処理で、1から `max` までの整数の合計を返す問題です。

例:

```text
max = 5
1 + 2 + 3 + 4 + 5 = 15
```

## 小学生レベルの言い換え

1から順番に数を足していく問題です。

```text
1を足す
2を足す
3を足す
4を足す
5を足す
```

全部足すと15になります。

## 変数の意味レイヤー

| 変数 | 意味 | イメージ |
| --- | --- | --- |
| `calcX` | 合計を入れておく箱 | 貯金箱 |
| `n` | 今足している数 | 今入れるお金 |
| `max` | 最後に足す数 | どこまで足すか |

`calcX = 合計箱`、`n = 今の数`、`max = 最後の数` と読むと追いやすくなります。

## 計算・処理の流れ

最初に、合計箱を空にしておきます。

```text
calcX <- 0
```

次に、`n` を1から `max` まで1ずつ増やします。

そのたびに、今の `n` を合計箱に足します。

```text
calcX <- calcX + n
```

## 代入の意味

`calcX <- calcX + n` は、普通の算数の等号ではありません。

プログラムでは、右側を計算して、左側に入れ直すという意味です。

```text
新しいcalcX = 古いcalcX + 今のn
```

代入は上書きです。左と右に同じ変数が出ても、前の値を使って新しい値を作る処理として読みます。

## 具体例

`max = 5` の場合:

| 回数 | n | やること | calcX |
| --- | --- | --- | --- |
| 最初 | - | 初期値 | 0 |
| 1回目 | 1 | 0 + 1 | 1 |
| 2回目 | 2 | 1 + 2 | 3 |
| 3回目 | 3 | 3 + 3 | 6 |
| 4回目 | 4 | 6 + 4 | 10 |
| 5回目 | 5 | 10 + 5 | 15 |

最後に `return calcX` なので、15を返します。

## 選択肢の切り方

| 選択肢の形 | 何が起きるか | 切り方 |
| --- | --- | --- |
| `calcX <- calcX * n` | 最初が0なので、ずっと0になりやすい | 掛け算なので合計ではない |
| `calcX <- calcX + 1` | 毎回1だけ足す | 回数を数えるだけ |
| `calcX <- calcX + n` | 今の数を合計へ足す | これが合計 |
| `calcX <- n` | 毎回nで上書きする | 最後はmaxが入るだけ |

合計したいなら、今の合計に今の数を足します。

## 誤答しやすい理由

`calcX <- calcX + n` が、普通の算数の式に見えてしまうと混乱します。

でもこれは、前の合計に今の数を足して、もう一度同じ箱へ入れる処理です。

## フロントエンド・実務との関係

JavaScriptでも同じ考え方を使います。

```js
let total = 0;

for (let n = 1; n <= 5; n++) {
  total = total + n;
}
```

ITパスポートの `calcX <- calcX + n` は、JavaScriptの `total = total + n;` に近いです。

資格暗記だけでなく、ループ、代入、合計処理の基礎としてフロントエンドにもつながります。

## 最後に見る一言

合計問題は、箱に今の値を足して戻す。

```text
合計処理 = 変数 <- 変数 + 今回の値
```
