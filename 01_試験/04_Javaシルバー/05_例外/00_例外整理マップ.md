# 例外整理マップ

## 目的

Java Silver の例外処理、`try` / `catch` / `finally`、チェック例外と非チェック例外を整理します。

## 問題で見る手がかり

- 例外の種類
- `try` / `catch` / `finally`
- `throw` / `throws`
- catch の順番
- 例外発生後の実行順

## まず切る選択肢

- 親例外を先に catch して子例外に届かないコード
- チェック例外の処理を忘れているコード
- `finally` の実行を見落としている選択肢
- `throw` と `throws` を混同している選択肢

## 似た構文との住み分け

- `throw` は例外を投げる
- `throws` は例外が外へ出る可能性を宣言する
- `finally` は通常終了でも例外発生でも基本的に実行される
- catch は子例外から親例外の順に置く

## 認知アプローチ

例外問題では、先にどこで例外が発生するかを見ます。次に、どの `catch` に入るか、`finally` がいつ実行されるかを追います。
