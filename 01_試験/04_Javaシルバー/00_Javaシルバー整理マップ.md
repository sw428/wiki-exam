# Javaシルバー整理マップ

## 目的

このフォルダは、Java Silver の文法、実行結果、型、例外、APIの安定知識を置く場所です。

Java Silver では、知識名よりも「コンパイルできるか」「実行時に落ちるか」「どの出力になるか」を先に見ます。

## 置くもの

- Java Silver の出題範囲に入る文法整理
- コンパイルエラーと実行時エラーの区別
- 型変換、継承、ポリモーフィズムの判断基準
- コード問題を読む順番
- 似た構文やAPIの住み分け
- [`02_検証/04_Javaシルバー`](../../02_検証/04_Javaシルバー/00_Javaシルバー検証マップ.md) から戻した安定知識

## 問題での基本判断

```text
構文が成立するか -> 型が合うか -> 初期化されているか -> 実行順はどうなるか -> 出力は何か
```

コード問題では、最初に正解を探さず、先に「コンパイルエラーで切れるか」を見ます。

用語の意味だけで判断せず、構文、型、スコープ、初期化、評価順をコード上で追います。必要な観点だけを選び、すべてのコード問題へ同じ説明見出しを強制しません。

## 処理の流れ

1. コード問題や誤答は、まず検証側に置く
2. 間違えた理由を「構文 / 型 / 継承 / 例外 / API / 実行順」に分ける
3. 次回も使う読み方だけを、このフォルダへ戻す
4. 分野が決まる内容は各カテゴリへ、横断する読み順はこの `00_` に置く

## 置かないもの

- 解きかけのコード問題ログ
- 実行結果だけのメモ
- Java Gold寄りの高度な整理

## カテゴリ追加ルール

- 必要なカテゴリは、この `04_Javaシルバー` フォルダ配下に作る
- カテゴリやファイルを追加するときは、`01_...` / `02_...` の連番をこのフォルダ内で合わせる
- 知識整理では、文法カテゴリ、実行順、エラー種別、似た構文やAPIの住み分けを優先する

## 今後の追加方針

新しい論点を追加するときは、次の形に寄せます。

```text
問題文・コードの手がかり
切るべきエラー種別
似た構文やAPIとの差
出力、例外、コンパイル可否の判断順
```
