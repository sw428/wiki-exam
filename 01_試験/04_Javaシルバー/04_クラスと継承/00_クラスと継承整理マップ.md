# クラスと継承整理マップ

## 目的

Java Silver のクラス、コンストラクタ、継承、オーバーロード、オーバーライドを整理します。

## 問題で見る手がかり

- コンストラクタ呼び出し
- `this` / `super`
- 継承関係
- オーバーロード、オーバーライド
- アクセス修飾子
- ポリモーフィズム

## まず切る選択肢

- コンストラクタ名や戻り値の扱いが誤っているコード
- `super()` の呼び出し位置が不正なコード
- オーバーロードとオーバーライドを混同している選択肢
- 参照型と実体型の違いを見落とした出力

## 似た構文との住み分け

- オーバーロードは同じ名前で引数が違う
- オーバーライドは親のメソッドを子が上書きする
- フィールドは参照型で見られやすく、メソッドは実行時の実体型で選ばれやすい
- `this` は自分、`super` は親を見る

## 認知アプローチ

継承問題では、参照型と実体型を分けます。次に、コンパイル時に見える範囲と、実行時に選ばれるメソッドを分けて追います。
