# 制御構文整理マップ

## 目的

Java Silver の `if`、`switch`、ループ、`break`、`continue` を整理します。

## 問題で見る手がかり

- 条件式の型
- `switch` のラベル
- `for` / `while` / `do-while`
- `break` / `continue`
- ループの初期値、条件、更新式

## まず切る選択肢

- 条件式として使えない型を使っているコード
- ループの更新タイミングを取り違えた出力
- `break` と `continue` の効果を混同している選択肢
- `do-while` が最低1回実行される点を見落とした選択肢

## 似た構文との住み分け

- `while` は条件を見てから実行
- `do-while` は実行してから条件を見る
- `break` はループや `switch` を抜ける
- `continue` は次の繰り返しへ進む

## 認知アプローチ

制御構文では、条件式よりも先に「何回実行される可能性があるか」を見ます。次に、更新式と抜ける条件を追います。
