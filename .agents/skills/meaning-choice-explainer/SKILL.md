# 語源・意味レイヤー四択判別

このスキルは、資格試験の四択用語問題で「言葉は見たことがあるのに、選択肢が全部それっぽく見える」状態をほどくために使います。

単なる正解暗記ではなく、用語の語源、普通の日本語としての意味、試験用語としての意味、問題文の手がかり語、選択肢の切り方まで分けて説明します。

## 使う場面

- 英単語、カタカナ語、漢字語の意味を分解すると見分けやすい問題
- 見積り、管理、対策、リスク、セキュリティなど、同じジャンルの用語が並ぶ問題
- 問題文中の手がかり語と、試験上の定義を対応させる必要がある問題
- 知識ゼロではなく、似た用語の違いで落としている問題
- 正解だけでなく、他の選択肢をなぜ切るかまで残したい問題

ITパスポート専用ではなく、必要に応じて基本情報技術者、応用情報技術者、Java Silver / Gold でも使います。

## 使わない場面

- 計算手順そのものが中心の問題
- コードの実行順やコンパイル可否が中心の問題
- 語源や言葉の意味より、仕様・公式・構文ルールを確認すべき問題
- 語源で説明すると、かえって覚え違いしそうな問題

語源や英単語の由来が不確かな場合は、歴史的事実のように断定せず、「意味をつかむための補助線」として扱います。最終判断は、必ず試験上の定義を優先します。

## 解説手順

1. 正解だけを先に出さず、問題文の手がかり語から順番に判断する
2. 問題が何を聞いているかを、小学生レベルで言い直す
3. 出題者が確認したいことを整理する
4. 正解になる用語を、語源・普通の意味・試験上の意味に分ける
5. なぜ四択が全部それっぽく見えるのかを説明する
6. どの言葉がどの用語につながり、どの言葉で他の選択肢を切れるかを書く
7. 問題文で線を引く場所を示す
8. 正解の選択肢と用語名を書く
9. 各選択肢について、何を言っているか、正解と何が違うか、一瞬で切る見分け語を説明する
10. 誤答しやすい思考パターンを書く
11. 本番で思い出すための短い暗記フレーズを作る
12. 小学生でも想像できる具体物のたとえで説明する
13. 次に同じタイプの問題を見たとき、何だけ見ればよいかを1文でまとめる

## 出力形式

次の見出しを使います。

- 問題文の要約（小学生レベル）
- 出題者の意図
- 用語の語源・意味レイヤー
- 四択が全部同じに見える理由
- 一段深い判別ポイント
- 問題の核心（線を引く場所）
- 正解
- 他の選択肢を切る理由
- 誤答しやすい思考パターン
- 一瞬暗記フレーズ
- 記憶固定イメージ
- 最後に一言

## 選択肢説明の形

各選択肢は、次の形で説明します。

```md
ア：
👉 これは〇〇。
👉 見分け語は「〜」。
👉 正解ではない理由は「〜」。
```

## 判別フレーズ例

- 画面・帳票・ファイル = ユーザーから見える機能の部品 = ファンクションポイント法
- ステップ・行数 = プログラム内部の量 = ステップ数見積り
- 類似・過去・似たシステム = 前例から考える = 類推法
- WBS・作業項目・分解・積み上げ = 作業を分けて足す = 積算法
