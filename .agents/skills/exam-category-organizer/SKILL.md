# 試験別カテゴリ整理

このスキルは、資格試験ごとの内容を、試験対応に使いやすいカテゴリフォルダへ整理するために使います。

目的は、知識をただ増やすことではありません。問題を見たときに「どのカテゴリの話か」「何と何を区別すればよいか」「どの手がかり語で選択肢を切るか」を再利用できる形にすることです。

## 基本方針

- 問題演習ログ、誤答、GPT回答の確認は、まず `02_検証/<資格>` で受ける
- 安定して使える知識だけを `01_試験/<資格>` に戻す
- `01_試験/<資格>` では、必要なカテゴリフォルダをその都度作る
- カテゴリフォルダは、各資格フォルダ内で `01_...` / `02_...` の連番を合わせる
- 新しいカテゴリを作ったら、対象資格フォルダの `00_...整理マップ.md` も同時に更新する
- 学習方法そのものは原則として残さず、問題演習・誤答分析・確認順に直接つながる検証だけを `02_検証` に残す
- `01_試験` には試験対応に使える知識整理だけを戻す

## 試験別のカテゴリ方針

### ITパスポート

最初に `ストラテジ / マネジメント / テクノロジ` を判定します。

- ストラテジ: 会社、経営、法務、知的財産、業務、システム戦略
- マネジメント: プロジェクト、サービス運用、障害対応、開発管理
- テクノロジ: 通信、コンピュータ、セキュリティ、データベース、基礎理論

ITパスポートでは、難しい説明を増やすより、問題文の短い手がかり語からカテゴリを切ることを優先します。

#### 短縮ではなく認知変換

ITパスポートの問題ログやGPT回答を整理するときは、単に短く要約することを目的にしません。

整理とは、次に同じタイプの問題を見たときに、問題文の手がかりから正解へ戻れるように、理解のレイヤーを挟んで再配置することです。

優先して残します。

- 正解だけでなく、なぜその答えになるか
- 問題文のどの言葉が手がかりになるか
- 選択肢をどう切るか
- 用語の普通の意味と、試験上の意味のズレ
- 小学生でもイメージできる具体物や日常例
- 暗記フレーズの前に置く意味の橋渡し
- 問題傾向として何を聞かれやすいか

避けます。

- 正解と一言暗記だけに圧縮する
- 判断理由が分からないほど説明を削る
- 教科書的な定義だけを書き、問題文の手がかり語と接続しない
- 用語を一覧化するだけで、似た選択肢との違いを書かない

ITパスポートの1テーマは、必要に応じて次のレイヤーから選びます。すべての見出しを毎回使う必要はありません。

```md
## 何の問題か
## 問題文の手がかり
## 小学生レベルの言い換え
## 試験上の意味
## 普通の言葉とのズレ
## 一瞬で思い出すイメージ
## 選択肢の切り方
## 誤答しやすい理由
## 計算・処理の流れ
## フロントエンド・実務との関係
## 最後に見る一言
```

問題タイプごとの優先は次の通りです。

- 用語問題: 意味、カタカナ語・英語・漢字の分解、試験上の定義、似た用語との違い、見分け語
- 計算問題: 何を求めるか、数字の意味、使う順番、途中状態の変化、選択肢との対応
- プログラム・擬似言語問題: 変数を箱として説明し、ループごとの値の変化、代入が上書きか追加かを書く
- セキュリティ問題: 何を守るか、検知・防御・許可・記録のどれか、ネットワーク側かWebアプリ側かを書く

### 基本情報技術者

科目Aと科目Bの違いを意識します。

- 科目A_基礎知識: 広い知識、用語、基本概念
- 科目B_アルゴリズム: 擬似言語、実行順、条件分岐、ループ
- 情報セキュリティ: 攻撃、対策、認証、暗号、運用
- ネットワーク・データベース: 通信、SQL、正規化、設計
- 計算問題: 進数、確率、稼働率、待ち行列など

### 応用情報技術者

午前知識と午後読解を分けます。

- 午前知識: 用語、定義、分野横断の基礎
- 午後問題の読み方: 設問要求、条件整理、根拠抽出
- セキュリティ: 重点分野として優先
- ネットワーク / データベース / マネジメント: 午後選択や頻出観点に合わせる

### Java Silver

文法と実行結果を重視します。

- 基本文法
- 型と変数
- 制御構文
- クラスと継承
- 例外
- 標準API

各カテゴリでは、コンパイルエラー、実行時エラー、出力結果、スコープ、評価順を区別します。

### Java Gold

APIと評価順を重視します。

- ラムダ式
- Stream API
- ジェネリクス
- 例外とリソース管理
- 並行処理
- I/O と NIO

各カテゴリでは、似たAPI、戻り値、遅延評価、例外発生位置、スレッド安全性を区別します。

## 出力の整理型

カテゴリ内のノートは、できるだけ次の型で整理します。

```md
## 目的

## 問題で見る手がかり

## まず切る選択肢

## 似た用語・似た構文との住み分け

## 認知アプローチ

## 次に見るリンク
```

## 判断基準

- 公式出題範囲よりも、試験で迷いやすい単位を優先する
- 既存の資格フォルダ内の連番と `00_` マップは必ず合わせる
- 迷ったら、問題文の手がかり語、誤答理由、選択肢の切り方を基準にカテゴリを決める
- 1ファイルが大きくなりすぎたら、同じカテゴリフォルダ内に連番ファイルを追加する
