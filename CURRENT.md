## 現在の状態

- `wiki-exam` は試験用 wiki として運用する
- root 直下は `ウィキのカスタムの仕方` / `01_試験` / `02_検証` の3本に整理済み
- `01_試験` 配下は、次の5フォルダに整理済み
  - `01_ITパスポート`
  - `02_基本情報技術者`
  - `03_応用情報技術者`
  - `04_Javaシルバー`
  - `05_Javaゴールド`
- `02_検証` は、問題演習・誤答・GPT回答をCodexが分解し、説明や判断方法の採用可否を整理する工程として運用する
- `01_試験` には普遍的知識だけでなく、同型問題の正答判断に再利用できると採用した認知変換・たとえ・確認順も、正式な定義と区別して戻す
- `02_検証` 配下にも、`01_試験` と同じ5つの試験別フォルダを作成済み
- 各フォルダに `00_...整理マップ.md` / `00_...検証マップ.md` を置き、フォルダの入口・処理ルール・リンク集として使う運用を復元済み
- 試験問題の解説とカテゴリ整理は `AGENTS.md` を共通の入口とし、試験固有の詳細は各資格・検証側の `00_` マップで確認する運用へ統合済み
- ITパスポート整理では、短く要約するより、問題文の手がかりから正解へ戻れる認知変換を優先する運用を追加済み
- root に `ウィキのカスタムの仕方` フォルダを作成し、`00_ウィキのカスタムの仕方.md` を入口、`01_元ウィキから資格ウィキへのカスタム手順ログ.md` を手順ログとして追加済み
- `CHANGELOG.md` は旧wiki由来の履歴を消し、試験用 wiki の恒久ログとして再開始済み
- `AGENTS.md` は、旧wikiの中核運用を戻しつつ、試験用フォルダ構成に合わせて更新済み

## この repo の今の目的

- ChatGPT / Codex / ユーザーが、試験ごとの現在地を再把握しやすい状態を作る
- 問題演習や誤答分析を、あとから再利用できる判断軸へ変換する
- 何を覚えるかだけでなく、問題文の読み方、選択肢の消し方、間違えた理由、次回の確認順を残す
- 試験対策に無関係な実務案件・制作記録・ポートフォリオ記録は持ち込まない
- 資格勉強に関係ない貼り付け内容は、引用・要約・保存・ログ化せず、関係ある部分だけを処理する

## 次にやること

1. 作業対象フォルダの `00_` ファイルを先に確認する
2. 問題演習や誤答分析は、まず `02_検証` の該当試験フォルダで受ける
3. 検証後、カテゴリ別知識、用語の住み分け、選択肢を切る認知アプローチ、同型問題で再利用できる認知変換を `01_試験` の該当試験フォルダへ戻す
4. 必要なカテゴリは各資格フォルダ配下に作り、その資格フォルダ内で連番を合わせる
5. フォルダ構成や処理ルールが変わったら、対応する `00_` ファイルも更新する
6. 運用変更があれば、`CHANGELOG.md` に追記する
7. 問題形式に応じた説明観点は `AGENTS.md` から選び、試験別の判断順は対象資格・検証側の `00_` マップで確認する
8. 貼り付けログに資格勉強と無関係な話題が混ざっている場合は、その部分に触れず、資格勉強に関係する部分だけ整理する

## 運用メモ

- フォルダ名とファイル名は、原則として `01_名前` / `02_名前` 形式にする
- 各フォルダの `00_` ファイルは、そのフォルダの入口・処理ルール・リンク集として扱う
- 各資格フォルダ配下では、必要になったカテゴリを作成してよい。その場合は同じ資格フォルダ内で連番を合わせる
- 1階層目（top直下）のフォルダは基盤として固定し、新規作成 / rename / move / delete は明示指示がある時だけ行う
- 2階層目以降は、導線改善や重複解消のために Codex が必要な整理を行ってよい
- delete は階層を問わず、明示指示がある時だけ行う
- Git 操作、特に `push` / `pull` / `remote` 変更は、ユーザーの明示指示がある場合だけ行う
