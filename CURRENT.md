## この repo の今の目的

- ChatGPT / Codex / ユーザーが、試験ごとの現在地を再把握しやすい状態を作る
- 問題演習や誤答分析を、あとから再利用できる判断軸へ変換する
- 何を覚えるかだけでなく、問題文の読み方、選択肢の消し方、間違えた理由、次回の確認順を残す
- 試験対策に無関係な実務案件・制作記録・ポートフォリオ記録は持ち込まない
- 資格勉強に関係ない貼り付け内容は、引用・要約・保存・ログ化せず、関係ある部分だけを処理する

## 次にやること

1. 作業対象フォルダの `00_` ファイルを先に確認する
2. 問題演習や誤答分析は、まず `02_検証` の該当試験フォルダで受ける
3. 検証後、安定した知識だけを `01_試験` の該当試験フォルダへ戻す
4. フォルダ構成や処理ルールが変わったら、対応する `00_` ファイルも更新する
5. 運用変更があれば、`CHANGELOG.md` に追記する
6. 用語の意味境界で四択が似て見える問題では、必要に応じて「語源・意味レイヤー四択判別」スキル（`$meaning-choice-explainer`）を使う
7. 貼り付けログに資格勉強と無関係な話題が混ざっている場合は、その部分に触れず、資格勉強に関係する部分だけ整理する

## 運用メモ

- フォルダ名とファイル名は、原則として `01_名前` / `02_名前` 形式にする
- 各フォルダの `00_` ファイルは、そのフォルダの入口・処理ルール・リンク集として扱う
- 1階層目（top直下）のフォルダは基盤として固定し、新規作成 / rename / move / delete は明示指示がある時だけ行う
- 2階層目以降は、導線改善や重複解消のために Codex が必要な整理を行ってよい
- delete は階層を問わず、明示指示がある時だけ行う
- Git 操作、特に `push` / `pull` / `remote` 変更は、ユーザーの明示指示がある場合だけ行う
