# wiki-exam

この wiki は、資格試験の学習内容、問題演習、誤答分析、確認順を整理するためのローカル Markdown wiki です。

元のフロントエンド実務 wiki からコピーして作っていますが、この repo では実務案件・制作記録・ポートフォリオ記録ではなく、試験対策だけを扱います。

## 使い方

試験ごとの安定した知識は `01_試験` に置きます。

問題演習、誤答分析、GPT回答の確認、テストに直接関係する確認順・覚え方の検証は、まず `02_検証` の該当試験フォルダで受けます。そこから再利用できる知識だけを、`01_試験` の該当試験フォルダへ戻します。

各フォルダの `00_...整理マップ.md` / `00_...検証マップ.md` は、そのフォルダの入口です。置くもの、置かないもの、処理の流れ、関連リンクはまず `00_` ファイルで確認します。

この wiki は、すべてを自力で暗記するための教科書ではありません。ChatGPT / Codex / 検索 / 問題集を併用しても、判断を外さないための外部脳として使います。

資格の整理では、短くすること自体を目的にせず、問題文の手がかり、意味の橋渡し、選択肢の切り方が残る形に変換します。

## 学習方針

このリポジトリでは、資格学習の内容を単なる暗記ノートとしてまとめるのではなく、GPT や Codex を活用して、用語・問題文・選択肢の意味を構造的に整理します。

学習では、基礎となる知識の積み重ねによって、理解のしやすさや記憶への残り方が大きく変わります。そのため、一見簡単そうに見える内容でも、前提知識が不足している場合は、用語の意味、問題文の意図、選択肢の違いを一つずつ確認します。

また、人によって学習経験や前提知識のレイヤーは異なるため、どの知識や認識が必要になるかも一律ではありません。このリポジトリでは、一般的な学習順序だけに合わせるのではなく、自分にとって不足している前提や、理解を進めるうえで必要な観点を確認しながら整理します。

重視するのは、答えを覚えることだけではありません。

- なぜその答えになるのか
- どこで誤解しやすいのか
- 自分に不足している前提は何か
- 問題文のどの言葉が手がかりになるのか
- 似た選択肢をどう切るのか

GPT や Codex は、作業を丸投げするためではなく、理解のズレを見つけ、学習内容を再整理するための補助として使います。

学習方法や努力の見え方は人によって異なります。そのため、外面的な作業量や一般的な方法論だけで理解度を判断するのではなく、目的に対して必要な知識、検証、時間を積み重ねることを重視します。

このリポジトリに記録している整理方法や考え方は、すべての人にそのまま当てはまる万能な方法ではありません。学習経験、前提知識、得意不得意、理解しやすい順序は人によって異なるため、自分の状態に合わせて内容を取捨選択し、必要に応じてカスタマイズすることが重要です。

また、学習において「楽をしているように見えるか」と「結果が出るかどうか」は、必ずしも直接結びつくものではありません。
重要なのは、手間をかけているように見えるかではなく、目的に対して必要な理解・検証・修正が適性に合ったうえで行われているかどうかだと考えています。

学習を進めるうえでは、適切に方向づけてくれる教育者や支援者が近くにいること、または自分で試行錯誤しながら傾向をつかむことが大切だと考えています。

特に、どこで理解が止まっているのか、どの前提知識が不足しているのかは外から見えにくいものです。そのため、実際に問題を解き、説明し直し、修正する過程を通して検証していきます。

## 記録するもの

- 試験ごとの重要語句
- 問題文の読み方
- 選択肢の消し方
- 間違えた理由
- 次回の確認順
- GPT回答を採用してよいかの判断
- 暗記ではなく理解の入口になる説明

## 記録しないもの

- 実務案件の詳細
- 制作記録
- ポートフォリオ記録
- 試験対策に関係しない個人話題
- 貼り付けログに混ざった資格勉強と無関係な話題
- 旧wiki由来の細かい実務運用
