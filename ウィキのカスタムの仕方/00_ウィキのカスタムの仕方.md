# ウィキのカスタムの仕方

## 目的

このフォルダは、元の `wiki` から `wiki-exam` へカスタムした流れ、判断理由、再利用できるプロンプト型を残す場所です。

元wikiの内容そのものではなく、別のwikiやワークスペースを作るときにも使える「カスタムの仕方」を記録します。

## ファイル

- [01_元ウィキから資格ウィキへのカスタム手順ログ](./01_元ウィキから資格ウィキへのカスタム手順ログ.md)
  - 元wikiを資格wikiへ変えたときのプロンプト、Codexの処理、最終構成、次回使えるプロンプト型を残す

## 置くもの

- 元wikiから別用途のwikiへ変える手順
- 残すシステムと削る内容の判断
- ラフなプロンプトから構成を調整した流れ
- 次回のカスタムに使えるプロンプト型
- Codexがどのファイルをどう変更したかの要約

## 置かないもの

- 試験そのものの知識
- 問題演習や誤答分析
- 実務案件・制作記録・ポートフォリオ記録の中身

## 処理の流れ

1. カスタムの元になるログやプロンプトを `01_...` 以降のファイルに置く
2. 再利用できる手順や判断軸を抽出する
3. wiki全体の運用に関わる内容だけを `AGENTS.md` / `CURRENT.md` / `CHANGELOG.md` へ反映する
4. 試験知識や検証ログは、このフォルダに混ぜず `01_試験` / `02_検証` に置く

## 現在の扱い

- このフォルダは番号付きトップカテゴリにせず、root 直下の補助フォルダとして残す
- 現在の試験フォルダ構成は README / CURRENT / `01_試験` / `02_検証` を正とする
- 手順ログ内の古いフォルダ番号は、その時点の会話ログとして残す
